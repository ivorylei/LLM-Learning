# 第 2 天 · 文字变数字 — 分镜脚本

> 给 Claude Code 的画面规格。口径忠于讲义第 2 天。风格见 CLAUDE.md 第 6 节。

---

## 镜 00 · 标题卡 `Day2Title`（~7s）
**画面**：深青底。大字"文字怎么变成数字"，暖橙副标题"意义即位置"。浅青小字："计算机只会算数，第一步永远是把字变成空间里的点。"
**旁白**：第二天。计算机只会算数字，不认识字。所以第一步永远是：把文字切成块，再把每一块变成空间里的一个点。今天的关键四个字——意义即位置。

---

## 镜 01 · 一条流水线 `Pipeline`（~20s）
**画面**：横向四格流水线，先只亮起标题，后面四镜逐个点亮：`分词` → `转 ID` → `词嵌入` → `位置编码`。一段中文/英文进入左端，右端流出一摞带编号的向量。
**旁白**：文字喂给模型前，要走一条四步流水线：先分词、再转成编号、再变成向量、最后加上位置信息。我们一步步来看。

---

## 镜 02 · 分词 `Tokenization`（~30s）
**画面**：
- 英文单词 "unbelievable" 被切成三块 "un | believ | able"（切割动画）。
- 一段中文也切成若干 token，提示"token 不等于字，也不等于词"。
- 标注 BPE：用有限的一套小块，拼出几乎任何词，包括没见过的新词。
**旁白**：第一步分词，把文字切成一个个小块，叫 token。token 不完全等于词——比如 unbelievable 可能被切成 un、believ、able 三块。这种切法叫 BPE，好处是：用有限的一套小块，就能拼出几乎任何词，包括从没见过的新词。

---

## 镜 03 · 转成 ID `TokenToID`（~20s）
**画面**：每个 token 旁边查一本"字典"，映射到一个整数编号。原句下方变成一串数字 [464, 1212, ...]。强调："文字到这里，就变成了一串数字。"
**旁白**：第二步，每个 token 去查模型内部的一本字典，得到一个整数编号，叫 token ID。到这一步，一句话就变成了一串数字。

---

## 镜 04 · 词嵌入：意义即位置 `Embedding`（~35s）
**画面**：
- 强调"编号 5 和编号 6 并不意味着意思相近"（划掉这种误解）。
- 每个编号映射成一长串小数（一个向量）。
- 切到一张 2D 语义地图：把"国王/女王""猫/狗""巴黎/伦敦"画成成对靠近的点；近义词聚成簇。字幕大字："意义，被编码成了空间里的位置。"
**旁白**：光有编号还不够——编号 5 和编号 6，并不代表意思相近。所以第三步，把每个编号再变成一长串小数，也就是一个向量，这叫词嵌入。神奇的是：训练之后，意思相近的词，它们的向量会自动靠得很近。国王和女王、猫和狗，在这个空间里就是邻居。意义，被编码成了空间里的位置。

---

## 镜 05 · 位置编码 `PositionalEncoding`（~25s）
**画面**：
- 两句对照："狗咬人" vs "人咬狗"——用的字一样，箭头表示语义相反。
- 给每个 token 方块标上序号①②③，表示"它排在第几位"的信息被加了进去。
**旁白**：但还差一样东西。"狗咬人"和"人咬狗"，用的字完全一样，意思却相反。所以第四步，要再给每个 token 加上"它排在第几位"的信息，这叫位置编码。走完这四步，一段文字就变成了一摞带位置的向量，可以喂给模型了。

---

## 镜 06 · 连接现实 `Day2RealWorld`（~35s）
**画面**（三个小卡片依次出现）：
- ①"上下文窗口 128K"= 一次最多读多少个 token；提示 token 数 ≠ 字数。
- ②按 token 收费 / 长对话会"忘事"：一个装 token 的容器装满后，最早的内容被挤出去。
- ③"strawberry 里有几个 r？"——模型看到的是切好的 token 块，不是一个个字母，所以会数错。
**旁白**：懂了今天的原理，很多怪现象一下子不神秘了。所谓上下文窗口 128K，是指模型一次最多能读多少个 token，而 token 不是字数。为什么按 token 收费、长对话会忘事？因为它一次能装的 token 有上限，超了就得丢掉最早的内容。为什么模型连 strawberry 里有几个 r 都数不清？因为它看到的不是一个个字母，而是被切好的 token 块——字母层面的信息，在分词时就被打包了。这不是 AI 变笨，是机制使然。

---

## 镜 07 · 收束卡 `Day2Outro`（~10s）
**画面**：深青底。大字"文字 → token → ID → 向量（空间里的点）"。下方小字呼应主线："这摞向量，就是去预测下一个词的原料。"
**旁白**：文字变成 token、变成编号、变成空间里的点。这摞带位置的向量，就是模型用来预测下一个词的原料。明天，我们进入整本书的心脏——注意力。

---

### 制作提示（给 Claude Code）

- 语义地图用 `Axes`/`NumberPlane` + `Dot` + 标签；成对词用相近坐标，连一条细线表示"相邻"。
- 容器"挤出最早内容"用一个矩形框 + token 方块队列 + 队首 `FadeOut`。
