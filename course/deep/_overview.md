> 这是 8 天概念课的**第二层**：代码与架构精讲册。
> 概念课负责「八个大想法」的直觉；本册负责**把书里每一段代码读懂、并能自己改着玩**。
> 目标读者仍是那个零编程基础、自学为主的 18 岁学生——只是这一次，我们不再绕过代码，而是**牵着手一行一行走过去**。

## 一、本册要解决的问题

概念课（`course/LLM底层原理_8天课程.md`）定了三条规矩，其中第一条是：

> **代码"读懂 + 跑通"即可，写不出来很正常。**

本册把这条规矩往前推**半步**——不是"能从零默写"，而是：

- **读懂**：书里每一段代码，知道它在干什么、为什么这么写、数据的形状怎么变。
- **能改**：能自己改参数、改一两行逻辑，预测结果会怎么变，跑一下验证。
- 为达到这两点，**在卡住的地方就地补 Python / PyTorch 基础**（不另开一门枯燥的语法课）。

**写不出整个类没关系，但每一行你都该能说出"它在算什么"。** 这就是本册的及格线。

## 四、每章统一模板（写每一章都套这个）

每个 `chNN_*.md` 固定这几块，顺序不变：

1. **章头卡**
   - 对应：概念课第 N 天 ｜ 书第 N 章
   - 一句话直觉（从概念课搬过来，保持口径一致）
   - **这一章你将亲手得到什么**（一个具体产物，例如"一个 `MultiHeadAttention` 类"）
   - **前置**：本章需要的 Python/PyTorch 基础清单（链接到 `ch00` 或就近的加油站）

2. **三分钟回顾**：用概念课的直觉，把这章要解决的问题再点一遍（不展开）。

3. **逐节精讲**（本册的主体，按书的小节走）。每个知识点用这个微循环：
   - **要解决什么**：一句话动机。
   - **代码**：贴书里的真实代码（**忠于书，不自创**；代码清单标注书中的清单号，如"代码清单 3.1"）。
   - **逐行拆**：一行一行说它在算什么、形状怎么变。
   - **🔧 亲手改一下**：给一个最小改动实验（改个数字/维度），让读者预测再验证——**这是"能改"的训练**。

4. **🐍 Python / PyTorch 加油站**（侧栏，穿插在需要处）：就地补一个语法点或 API（如 `@` 矩阵乘、广播、`dim=-1`、`nn.Linear`、`.view/.transpose`、`register_buffer`）。**只补这一段代码真正用到的**，不堆语法。

5. **练习**：
   - **书上原题**（标注题号，如"练习 3.1"）+ 解题思路 + 折叠的答案要点。
   - **本册自测题**（1–3 道"改了会怎样"的小实验题）。

6. **本章小结**：3–6 条带走的话。

7. **能改自检清单**：若干条"我能不能独立做到 X"。全勾 = 这章过关。

8. **通往下一章**：一句话承上启下。

> **写作风格**：像概念课一样口语、克制、抓主线（"预测下一个词"）。代码块用 ```python；张量形状用 `[6, 3]` 这种写法随手标注；术语第一次出现给中英对照。
