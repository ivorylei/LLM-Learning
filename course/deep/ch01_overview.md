## 0. 三分钟回顾（概念课直觉）

概念课第 1 天三件事，先攥在手里：

1. **整个领域只有一件事：预测下一个词。** 给一段文字，猜下一个最可能的词；把这个词接上去，再猜下一个，循环往复——这就是"回答"。
2. **造 LLM 分两阶段**：预训练（烧钱，学会语言和知识）→ 微调（便宜，学会听话）。
3. **都是同一个家族**：GPT/Claude/Gemini/DeepSeek… 底层都是 **Transformer**。搞懂书里这个小 GPT，就懂了它们的骨架。

本册要做的，就是把第 3 句话**坐实成代码**：真的把那个小 GPT 拼出来、跑起来。

## 1. 我们要造的机器：一张端到端流水线

把"预测下一个词"摊开成代码，是这样一条流水线：

```
       一段文字
          │  ① 分词 + 嵌入（第 2 章）
          ▼
   一摞带位置的向量  [批, 词数, 维度]
          │  ② 注意力（第 3 章）——词之间互相看
          ▼
   每个词的"上下文向量"
          │  ③ Transformer 积木 ×N（第 4 章）——注意力+前馈+残差+归一化，重复堆
          ▼
   最终每个位置的向量
          │  ④ 输出层
          ▼
   下一个词的概率分布  →  挑一个词  →  接回去，再来一遍
```

**每一章造哪一块，对得很整齐**：

| 本册章 | 书章 | 造出的零件 | 概念课 |
|---|---|---|---|
| `ch02` | 第 2 章 | **入口**：文字 → token → 嵌入向量（带位置）| 第 2 天 |
| `ch03` ✅ | 第 3 章 | **心脏**：`MultiHeadAttention`（你已经造好了）| 第 3 天 |
| `ch04` | 第 4 章 | **整机**：把心脏装进 Transformer 积木，堆 N 层成 GPT | 第 4 天 |
| `ch05` | 第 5 章 | **训练**：让它真的学会预测下一个词 + 加载 GPT-2 权重 | 第 5 天 |
| `ch06` | 第 6 章 | **改用途**：把 GPT 改成垃圾邮件分类器（微调）| 第 6 天 |
| `ch07` | 第 7 章 | **调行为**：指令微调，把"续写机"变"问答助手" | 第 6 天 |

> 看出来了吗？**第 3 章的注意力你已经造完了——那是最难的一块。** 剩下的是把它"装进整机"再"训练/改造"。施工已经过了最陡的坡。

## 2. 两阶段，在代码里分别是哪几章

概念课的 `海量文本 →（预训练）→ 基础模型 →（微调）→ 助手`，落到本册：

- **搭骨架**（第 2–4 章）：把模型结构造出来。此时它还没训练，吐出来是乱码，但**架构是真的、完整的**。
- **预训练**（第 5 章）：用"猜下一个词"把半个互联网压进参数。这一步极贵，所以书也教你**直接下载 OpenAI 的 GPT-2 权重**——省得自己烧钱。
- **微调**（第 6–7 章）：在训好的大脑上，用少量数据改行为（分类 / 听指令）。便宜、见效快。

**一句话**：第 2–4 章造"空大脑"，第 5 章给它"灌知识"，第 6–7 章教它"懂事"。

## 4. 一次最小的"跑起来"

不用等到第 2 章。现在就在 Colab（或本机）感受一下"运行代码、看到张量"：

```python
import torch
print("PyTorch 版本:", torch.__version__)

# 造一摞"假装是 3 个词、每个 4 维向量"的张量——LLM 吃进去的就是这种东西
x = torch.rand(3, 4)
print(x)
print("形状:", x.shape)          # torch.Size([3, 4])
```

跑出来你会看到一张 3×4 的小数表——**这就是"文字变成数字"之后，模型眼里的世界**（细节是第 2 章的事）。重点只有一个：**体验"点运行→出结果"，并习惯随手 `print(x.shape)`**。

> **预告一眼"整个模型有多大"**：第 4 章你会用一个配置字典定义整个 GPT-2 小模型，长这样（先混个眼熟，不用懂）：
> ```python
> GPT_CONFIG_124M = {
>     "vocab_size": 50257,    # 词表大小（第 2 章）
>     "context_length": 1024, # 一次最多看多少 token
>     "emb_dim": 768,         # 每个词多少维
>     "n_heads": 12,          # 多头注意力的头数（第 3 章）
>     "n_layers": 12,         # 堆多少块 Transformer 积木（第 4 章）
>     "drop_rate": 0.1,       # dropout（第 3 章）
>     "qkv_bias": False,
> }
> ```
> **整个 GPT 的"蓝图"就这么几行数字。** 你已经认识其中一半了（`n_heads`、`qkv_bias`、`drop_rate` 都是第 3 章的）。

## 5. 读这种代码的三个习惯（贯穿全册）

零基础读代码不慌的秘诀，就三条：

1. **先 `print(x.shape)`，再想逻辑。** 形状对了，逻辑八九不离十。第 3 章你已经体会到——盯住形状，绕的代码也能拆开。
2. **一块块跑，不要整页跑。** Colab 的格子就是为这个设计的：跑一格、看结果、确认懂了，再跑下一格。看不懂就把中间变量 `print` 出来。
3. **改一个数字，看它怎么变。** 把维度、层数、学习率改一个，预测结果会怎样，再跑验证。**"能改"就是这么练出来的**——本册每节的 🔧 都在逼你做这件事。

> 这三条不是技巧，是**心态**：你不是在"默写代码"，是在"拆解一台机器、拧拧它的螺丝看会怎样"。
