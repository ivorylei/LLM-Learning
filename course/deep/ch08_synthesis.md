## 1. 一张能装下整个 AI 世界的地图：四个抽屉

任何一个 AI 新进展，几乎都属于下面四类之一。看到任何新词，先问"它属于哪个抽屉"：

| 抽屉 | 它动的是 | 对应你读过的代码 |
|---|---|---|
| **① 更大/更好的预训练器** | 模型本身和它的训练 | [第4章](ch04_gpt.md)堆叠 + [第5章](ch05_pretrain.md)预训练 |
| **② 预训练后塑造行为** | 微调 / 对齐 | [第6章](ch06_classify.md)分类微调 + [第7章](ch07_instruct.md)指令微调 |
| **③ 推理时给更多上下文/工具/思考** | 不改模型，改"怎么用它" | [第2章](ch02_text_data.md)上下文 + [第3章](ch03_attention.md)注意力 |
| **④ 效率技巧** | 让模型更小/更快/更便宜 | [附录D](appD_training_tricks.md)训练技巧 + [附录E](appE_lora.md) LoRA |

## 2. 热词归位表：每个词，拆回你读过的那段代码

下面把当下最热的词逐个归位——不只说"属于哪个抽屉"，还指出**它动了你亲手读过的哪个零件**。

| 热词 | 抽屉 | 用你读过的代码，它其实是…… |
|---|---|---|
| **规模法则 / 更大的模型** | ① | 同一块 Transformer 积木（[第4章](ch04_gpt.md)）堆更多、喂更多数据、训练更久（[第5章](ch05_pretrain.md)），损失可预测地下降。 |
| **MoE（专家混合）** | ①④ | 把 [第4章](ch04_gpt.md) 的 `FeedForward` 换成很多个并排的小专家，每个 token 只走其中几个——参数极大但计算不成比例增加。 |
| **RLHF / 对齐** | ② | 在 [第7章](ch07_instruct.md) 的指令微调（SFT）之后，再用"人类更喜欢哪个回答"的偏好信号继续调教。机制还是梯度更新，只是信号换成了"偏好"。 |
| **推理模型（o 系列 / DeepSeek-R1）** | ②③ | 训练/引导模型在回答前先生成一长串"思考草稿"，把更多计算花在推理时。本质还是 [第5章](ch05_pretrain.md) 的解码循环——只是先预测一大段思考。 |
| **RAG（检索增强）** | ③ | 不把知识全塞进参数，而是临时检索相关文本、塞进上下文窗口（[第2章](ch02_text_data.md)），让 [第3章](ch03_attention.md) 的注意力去读它。 |
| **智能体（Agent）** | ③ | 把模型放进一个循环：输出"动作"→执行→把结果喂回上下文→再预测下一步。仍然是 [第5章](ch05_pretrain.md) 那个生成循环，只是这次预测的 token 是"动作"。 |
| **多模态（看图、听声）** | ①③ | 把图像/音频也切成 token、变成向量（[第2章](ch02_text_data.md) 的嵌入思路），和文字一起喂进同一个模型。一切皆向量。 |
| **长上下文（128K / 1M）** | ④ | 想办法绕开 [第3章](ch03_attention.md) 注意力 `attn_scores = queries @ keys.T` 那个 n×n 的成本，让模型一次能读更多 token。 |
| **量化 / 蒸馏** | ④ | 量化＝用更少的位数存参数（参数本就是 [第4章](ch04_gpt.md) 里那一堆数字）；蒸馏＝训练一个小模型去模仿大模型。都是为了更小更快。 |
| **LoRA / PEFT** | ②④ | 冻结原模型，只在每个 `nn.Linear` 旁挂一对低秩矩阵 A·B 来训练——你在 [附录E](appE_lora.md) 已经把它接到第6章的分类微调上跑过。 |
| **训练技巧（warmup/cosine/clip）** | ①④ | 给 [第5章](ch05_pretrain.md) 训练循环加的小补丁，让大模型训得更稳——就是 [附录D](appD_training_tricks.md) 那三招。 |
| **开源 vs 闭源** | 全部 | 用的是同一套配方。开源能追平，正因为底层原理没有秘密——而这套原理，你已经**从第2章到附录E逐行拆过了**。 |

> 注意到了吗？整张表里**没有一个词逃出你读过的代码**。这不是巧合——这就是"搞懂书里这个小模型，就懂了所有大模型骨架"的真正含义。

## 3. 读新闻的三个追问

以后刷到任何 AI 新闻，先别管它包装得多唬人，问自己三个问题：

1. **它属于哪个抽屉？**（做大模型、改调教、改用法、还是求效率？）
2. **它动了我读过的哪段代码？**（嵌入？`MultiHeadAttention`？`FeedForward`？训练循环？输出头？解码？）
3. **它是真·新原理，还是在已知主干上加的一根枝？**（99% 是后者。）

能稳定回答这三个问题，你对 AI 的认识就已经是"根本性"的了——**不被热词裹挟，能看见骨架。** 区别在于：现在你回答第 2 问时，脑子里浮现的不是一个名词，而是一段你真正读懂、还能改的代码。

## 4. 结课作业（Capstone，建议认真做）

不写新代码，做一件更有价值的事：

> **找三条最近一周的 AI 新闻/产品发布，每条用第 1–3 节的方法分析一遍**——它属于哪个抽屉、动了你读过的哪段代码、是新原理还是加的枝。写下来。

加分项（把"能改"用起来）：挑其中一条，去对应章节的 Colab notebook 里，**改一个相关的参数或一两行代码**，亲眼看看它对应的零件怎么变。比如读到"长上下文"，就回 [第3章](ch03_attention.md) 把 `context_length` 调大，看注意力矩阵怎么变大；读到"MoE"，就回 [第4章](ch04_gpt.md) 看看 `FeedForward` 长什么样、想象把它换成并排的几个。

能把这三条讲清楚、并动手验证一次，你就真正完成了这门课的目标：**不只是知道 LLM 怎么造，而是获得了看懂整个 AI 世界的底层视角。**

## 5. 在飞速变化的领域里，怎么保持判断（送你的话）

这个领域几乎每周都有新模型发布，信息多到让人焦虑。但你现在有了"免疫力"：

- **底层原理变化极慢**——Transformer、注意力、预训练+微调，这套骨架好几年没变。你学的是骨架，不是会过期的新闻。
- **新东西几乎都是已知零件的重新组合**。你已经有了零件清单——而且是**逐行读懂、能上手改**的那种清单。
- **判断一个新进展重不重要，就看它在哪个抽屉、解决了什么真问题**，而不是看它的发布会多热闹。
